# WAI Pattern — The Work Contract

A **pattern** is the contract: the arrangement of lugs (holes on the wheel) as a versioned, verifiable checklist. The **bolt** is the torque-certification fastened through a closed pattern — it certifies every item is non-hallucinated. No bolt until the pattern checks out true.

> Schema: `schemas/pattern.schema.json`. Bolt: `schemas/bolt.schema.json`. Lug lease/verify fields: `wai-lug-schema.md`.

---

## Mental model

- **Lug = WHAT** (a hole / deliverable). **Initiative = WHEN** (grouping/timing). **Session = WHO**. **Pattern = the contract** binding a set of lugs with their verifications. **Bolt = the certification** of a closed pattern.
- A pattern formalizes what an epic's `build_waves` did informally — but with per-item verification and a certification gate.

## Authoring a pattern

Create `WAI-Spoke/patterns/bytype/pattern/draft/{id}.json` (`id = pattern-{slug}-v{n}`):

1. **items[]** — one entry per lug in the pattern. Each declares how it is verified:
   - `verify.mode: mechanical` — `verify.assertion` is a runnable check (command exits 0 / file exists / test passes / schema validates).
   - `verify.mode: attested` — `verify.verifier` names the AI verifier role expected to sign (e.g. `lug-reviewer`).
   - `verify.mode: human` — surfaced to Mario's queue to sign.
2. **preconditions[]** — assertions that must hold before the pattern may be actioned (required state + inputs available). The verify-before-action gate evaluates these.
3. **provenance** — the inputs that manifested this pattern: `input_lugs`, `input_teachings`, `input_patterns`, `derived_from` (prior version), `state_hash`. This is the versioned DNA — it makes the pattern's origin replayable.
4. **lifecycle_state** — start `draft`.

## Lifecycle

```
draft → active → certified | failed | superseded
```

- **draft → active:** when the first lug in the pattern is leased (work begins). Move the file to `active/`.
- **active → certified:** every item verified `pass`. Set `certified_at`, `bolt_id`, move to `certified/`.
- **active → failed:** any item's verification fails. Move to `failed/`; the failure is recorded in the bolt.
- **→ superseded:** a new pattern version replaces this one (`provenance.derived_from` on the new version points back). Move to `superseded/`.

## Closing a pattern (emits the bolt)

**Closing happens at BOTH `/wai-savepoint` and `/wai-closeout`** — closing a pattern is the finishing act.

1. For each item, run its verification per `verify.mode` (the tiered verify engine in `tools/` runs mechanical assertions, routes attested items to the named verifier, queues human items).
2. Record per-item `result {verified_by, verified_at, pass}` and set item `status`.
3. Emit a **bolt** (`schemas/bolt.schema.json`):
   - **certified** — every item passed. Pattern → `certified`.
   - **partial** — pattern closed early (savepoint, or some items still `pending`/unverified). The bolt records which items are certified and which remain, so the next worker **resumes from proof**, not archaeology.
4. The bolt carries the pattern's `provenance` forward — the certification is itself a node in the DNA lineage.

## Rules

- A pattern is not certified until **every** item passes its declared verification. Nothing passes uncertified — `human`/`attested` items still produce a recorded, accountable signature.
- Closing at savepoint means interrupted/paused sessions still leave a certified (or partial) receipt — no journey holes.
- `WAI-PatternIndex.jsonl` is generated by the Basher index tooling (walk `WAI-Spoke/patterns/bytype/**`) — do not hand-maintain it.
