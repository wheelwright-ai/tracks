# Crew Track Schema — per-advisor `track.jsonl` event vocabulary

**Status:** Living document. Source of truth for crew event logging.
**Owner:** Dana (delivery-manager).
**Pairs with:** `templates/commands/wai-crew-convention.md` (folder layout)
and `templates/commands/wai-crew-honesty.md` (Honesty Contract).

Each crew advisor writes append-only events to its own
`crew/{slug}/track.jsonl`. Dana scans these to detect no-shows and surface
stalls; Historian sweeps them for retrospectives. This document defines the
record shape, the controlled event vocabulary, and the README hoist
convention.

---

## Record Shape

Every line of `crew/{slug}/track.jsonl` is a single JSON object:

```json
{
  "ts": "2026-05-22T14:32:00Z",
  "event": "lug_completed",
  "lug_id": "impl-wayfinder-cycle-close-helper-v1",
  "phase": "implement",
  "decision": null,
  "note": "All 4 verifies PASS"
}
```

**Required fields** on every event:

| Field | Type | Notes |
|-------|------|-------|
| `ts` | string | ISO-8601 UTC, with `Z` suffix. |
| `event` | string | One of the controlled vocabulary below. |

**Optional fields** (used by some events):

| Field | Type | Notes |
|-------|------|-------|
| `lug_id` | string | Stable lug ID this event references. |
| `phase` | string | One of: `ideate`, `design`, `implement`, `build`, `verify`, `observe`. |
| `decision` | string | Free text — only required by `decision_recorded`. |
| `note` | string | Free text — short context, one line. |

Unknown fields are tolerated (forward compatibility) but never required.

---

## Controlled Event Vocabulary

| Event | Required fields | Optional fields | Meaning |
|-------|-----------------|-----------------|---------|
| `phase_entered` | `ts`, `event`, `phase` | `note` | Advisor opened the named phase as lead. |
| `phase_exited` | `ts`, `event`, `phase` | `note`, `decision` | Advisor closed the phase. `decision` carries the gate outcome. |
| `lug_claimed` | `ts`, `event`, `lug_id` | `phase`, `note` | Advisor picked up a lug from incoming/. |
| `lug_completed` | `ts`, `event`, `lug_id` | `phase`, `note` | Advisor finished and verified a lug. |
| `lug_blocked` | `ts`, `event`, `lug_id`, `note` | `phase` | Advisor cannot proceed; `note` names the blocker. |
| `consult_requested` | `ts`, `event`, `lug_id`, `note` | `phase` | Lead asked a consultant for input on the named lug. |
| `consult_answered` | `ts`, `event`, `lug_id`, `note` | `phase` | Consultant replied. `note` summarises the response. |
| `no_show_flagged` | `ts`, `event`, `phase`, `note` | `lug_id` | Dana-only — recorded when a phase closes without a lead track entry. |
| `stall_surfaced` | `ts`, `event`, `lug_id`, `note` | — | Dana-only — recorded when an incoming lug exceeds the 24h SLA. |
| `decision_recorded` | `ts`, `event`, `decision` | `lug_id`, `phase`, `note` | A gate decision was made (e.g. risk_tier set, rollback approved). |
| `escalated_to_dana` | `ts`, `event`, `note` | `lug_id`, `phase` | Advisor escalated a dispute or blocker to Dana. |
| `version_bumped` | `ts`, `event`, `note` | — | Advisor's `version:` frontmatter changed; `note` cites the semver delta. |

Adding a new vocabulary term is a minor version bump of this document and
requires Dana's approval.

## Write Rules

1. **Append-only.** Never edit an existing line. Corrections happen via a
   follow-up event whose `note` references the prior `ts`.
2. **One event per acted session, minimum.** Any session in which the
   advisor was active MUST produce at least one event. Silence equals
   no-show.
3. **No batched timestamps.** Each event carries the `ts` it was actually
   written. Do not back-date.
4. **Rotation at 5000 lines.** When a `track.jsonl` exceeds 5000 lines,
   rename to `track-archive-{YYYY-MM}.jsonl` (current year-month) and start
   a fresh `track.jsonl`. Archive files remain readable but are not hoisted.

## Last-20 README Hoist

Each `crew/{slug}/README.md` MUST contain a heading
`## Recent Activity (last 20)` whose body is a fenced markdown table of the
20 most recent events from `track.jsonl`. The table is regenerated by:

```bash
python3 tools/crew_hoist.py {slug}
```

(The `crew_hoist.py` script is out of scope for this lug — it will be built
when the first advisor folder needs a refresh. Until then, hoist by hand
following the table shape below.)

Table shape:

| ts | event | lug_id / phase | note |
|----|-------|---------------|------|
| 2026-05-22T14:32:00Z | lug_completed | impl-wayfinder-cycle-close-helper-v1 | All 4 verifies PASS |

The README replaces the section in place — never appends a second copy.

## Worked Example

A complete five-event sequence covering one work cycle, ideate → observe:

```json
{"ts": "2026-05-22T09:00:00Z", "event": "phase_entered", "phase": "ideate", "note": "Pete drafting intent for caching layer"}
{"ts": "2026-05-22T10:30:00Z", "event": "phase_exited",  "phase": "ideate", "decision": "scope: read-side only", "note": "handing to Archie"}
{"ts": "2026-05-22T11:00:00Z", "event": "phase_entered", "phase": "design", "note": "Archie evaluating blast radius"}
{"ts": "2026-05-22T13:00:00Z", "event": "consult_requested", "lug_id": "design-cache-layer-v1", "phase": "design", "note": "asked Sage for secrets-handling review"}
{"ts": "2026-05-22T15:45:00Z", "event": "lug_completed", "lug_id": "impl-cache-layer-v1", "phase": "verify", "note": "Jordy: 12/12 tests pass"}
```

Each line is independently parseable; reading the file gives the full
sequence of an advisor's actions over time.
